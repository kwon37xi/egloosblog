~ 2010년 12월 20일까지의 기술 문서 정리

내 개인 위키를 장만했는데, 위키에만 너무 글을 쓰다보면 이미 소홀해질 대로 소홀해진 블로그가 너무 죽을까봐(이미 너무 죽었지?) 위키에 정리한 기술 문서들을 가능하면 일주일 정도의 단위로 정리하여 블로그에도 올려보기로 했다. 위키 내용 모두는 아니고 되도록 프로그래밍과 컴퓨팅 관련된 것만 정리한다.

한 주간 한 일들을 되돌아 보는 기회 정도가 되어 줄 수 있을 것 같은데, 하지만 얼마나 오래갈지는 모르겠다.

  • 이미지 축소시 원본 비율을 유지하면서 축소 크기보다는 항상 작은 크기를 구하는 방법과 함께
  • 이미지 축소시 축소할 크기에 맞게 원본 이미지를 잘라내는(crop) 크기를 구하는 방법을 정리 했다.
  • 내가 쓰는 Chrome 브라우저 확장 목록을 정리하고 있다. 현재 기본 브라우저로 Chrome을 사용하고 있다.
  • Eclipse에서 Tomcat 대신 Jetty를 사용하는 방법이 있다. 그리고 더욱 놀라운 것은 Jetty가 Eclipse 프로젝트로 편입 되었다.
  • Eclipse에서 JUnit으로 테스트를 작성할 때 static import 구문을 편리하게 만드는 방법을 정리하였다. 이는 Toby님의 "테스트 코드에서 static import를 편하게 넣는 방법"이라는 글에서 힌트를 받아서 더욱 발전 시킨 것이다. 내가 정리한 방법으로 하면 에디터의 어느 위치에서도 항상 static import를 Java파일의 import 영역에 넣어줄 수 있게 된다.
  • Java에서 문자열 중 정규표현식(regular expression)에 매칭되는 영역만 다른 문자열로 바꿔치기 할 때 역슬래시(\)와 달러 기호($)에서 발생하는 문제와 해결책을 정리하였다.
  • Java에서 링크 문자열(http://*)을 찾아서 실제 <a>태그로 감싸는 정규표현식
  • Ubuntu Linux 에서 터치패드의 두 손가락 스크롤 기능 활성화 방법
  • Linux에서 어떠한 애플리케이션이라도 시스템 트레이 아이콘으로 넣어 주는 프로그램
  • Dokuwiki 설치기

더 있긴 한데.. 이 정도면 제일 중요한 것들은 다 정리한 것 같다.
