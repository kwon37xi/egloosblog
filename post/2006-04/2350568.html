[Cygwin] Java 사용하기

Cygwin에서 Java를 사용할 때는 경로 지정 때문에 문제가 발생한다.

cygwin 에서는 Unix 처럼 /path/target 형태의 경로를 사용하지만, 거기서 실행하는 Java 는 Cygwin 어플리케이션이 아니라 윈도우용 어플리케이션이기 때문에 당연히 유닉스 방식의 경로 지정을 인지하지 못한다.

클래스패스를 지정할 때는 다음 처럼 먼저 Cygwin 방식으로 클래스 패스를 설정한 뒤에, 그것을 윈도우의 절대 경로로 바꾸면 된다. cygpath -wp 명령을 사용한다.
CYG_CLASSPATH=/home/kwon37xi/study_unittest/bin
CYG_CLASSPATH=$CYG_CLASSPATH:/home/kwon37xi/study_unittest/lib/junit.jar
CLASSPATH=`cygpath -wp $CYG_CLASSPATH`
export CLASSPATH

"echo $CLASSPATH" 해보면 유닉스 형태가 아닌 윈도우 형태의 경로로 설정돼 있음을 확인할 수 있다. 또한 ":"이던 경로 분리자가 윈도우 방식의 ";"로 자동으로 바뀐다.

javac -d 경로명령으로 컴파일 할 때는 저게 안 먹는다. cypath -wp 옵션은 C:cygwinhome.. 이런식으로 경로를 생성하는데, javac -d 에서 지정하는 경로는 Java 문자열 처럼 를 특수문자로 간주한다. 그럴때는 대신 / 를 사용하면 된다.
CYG_TARGET_DIR=/home/kwon37xi/study_unittest/bin
TARGET_DIR=`cygpath -mp $CYG_TARGET_DIR`

alias javac="javac -d $TARGET_DIR"

javac 명령을 실행하면 자동으로 -d 옵션이 붙는다.
