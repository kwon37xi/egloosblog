Python 3000 현재 진행 상황

귀도 반 로섬이 Python 3000 현재 진행 상황에 대한 글을 올렸다.

Py3k가 과연 나올 수 있을지 의문이라는 글을 몇개 읽은 적이 있었는데, 나오긴 나올 듯 하다. 올 8월말에 알파 버전 형태로 나오고 내년 말이 되면 정식이 나오지 않을까 싶다.

기본 소스 인코딩이 UTF-8이 될 것이고, 내부적으로는 Java와 동일하게 모든 문자열을 유니코드로 다룰 것이며, 변수명 등으로 유니코드 문자(즉, 한글도!) 쓸 수 있게 될 예정이다. 이에 따라 IO 도 Java 처럼 인코딩을 지정해서 수행할 수 있게 된다.

2.x 기반의 소스를 Py3k 기반으로 변경하는 방법에 관해서도 설명한다.

Py3k가 나오기 전에 Python 2.6이 나올 예정이며, 2.6에서 Py3k의 기능을 맛볼 수도 있을 것이다.
