Eclipse 3.6 Helios 이상 Java WebApp의 의존 프로젝트/Jar가 배포 안되는 문제

Eclipse 3.6 Helios 이상 버전에서 자바 웹 애플리케이션(WTP 플러그인 사용)을 톰캣에 올려 실행할 때 의존하는 프로젝트의 클래스 Jar 가 배포 안되는 문제가 발생하고 있다.

버그인 것 같으며, 이와 같은 문제는 Maven, Ivy 같은 의존성 관리 플러그인을 사용할 경우에도 나타날 가능성이 있는것으로 보인다.

 

이런 일이 발생하면 웹 프로젝트의 Project Properties -> Deployment Assembly -> [Add...]를 통해서 의존 프로젝트와 기타 Jar 들을 명시적으로 배포하라고 지정할 수 있다.

 

"Deployment Assembly" 는 과거 버전의 "J2EE Module Dependency" 메뉴가 바뀐것이라고 한다.

그게 바뀌었든 안 바뀌었든 의존 프로젝트를 제대로 배포해주지 못하는 것은 버그라고 밖에 볼 수 없다.
