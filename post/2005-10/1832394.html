[Eclipse] MyEclipseIDE 4.0 JSP->Java 변환된 소스 디버깅

Eclipse 3.1 + MyEclipseIDE 4.0 + Tomcat 으로 JSP 개발을 하는 도중 JSP에서 예외가 발생하면 JSP가 Java로 변환된 소스의 줄 번호가 예외 메시지에 나오게 된다. (JSP Spec 2.0/Servlet Spec 2.4 일 경우에는 JSP 줄수도 나온다)

이럴때 소스 디렉토리에 가서 읽는 방법도 있지만, 그러지 말고 아예 톰캣이 생성하는 Java 소스를 프로젝트 디렉토리에 넣고 그 디렉토리를 소스 디렉토리로 지정해 버리면 예외 메시지에서 클릭만 해도 소스로 간다.

$CATALINA_HOME/conf/Catalina/localhost/컨텍스트명.xml 을 만들고,
<Context path="/컨텍스트명" docBase="프로젝트실행디렉토리경로"
    debug="0" reload="false" workDir="프로젝트디렉토리/work">
</Context>

위와 같이 하면 톰캣이 JSP를 Java 로 변환한 소스를 프로젝트디렉토리/work에 저장하게 된다.

* Eclipse로 가서 work 디렉토리를 소스 디렉토리로 지정하고, class 내보내는 경로를 work 디렉토리 자신으로 지정한다.
: 프로젝트이름에서 오른쪽클릭 -> Properties -> Java Build Path -> Add Folder -> work 디렉토리 선택
: Allow output folders for source folders 선택
: work 디렉토리의 Output 폴더를 자기 자신으로 변경 work 디렉토리의 Output 폴더를 일반 자바 소스의 클래스 디렉토리와 전혀 상관없는 다른 디렉토리로 변경한다. work 디렉토리 자체로 Output시키면 안된다. Eclipse가 work 디렉토리의 소스를 강제로 컴파일하면서 문제를 일으킬 수 있다.

* work 디렉토리에서 오른쪽 클릭해서 이 디렉토리의 문자 인코딩을 UTF-8로 변경한다.

* $CATALINA_HOME/common/lib 디렉토리의 jsp-api.jar와 jasper-runtime.jar 를 클래스 패스에 추가한다.

* JSP를 실행하다 예외가 발생하면 work 디렉토리에서 F5를 눌러 변경사항을 읽게 한뒤에 예에 메시지의 JSP가 Java 로 변환된 소스 코드 부분을 클릭하면 그 부분으로 이동한다.

이제 쪼끔이나마 편하게 개발한 수 있다.

단... 나는 JSP 2.0/Servlet 2.4를 이용하는 것이 위와 같이 하는 것보다 정신 건강에 도움이 되리라 믿는다.
