Java Class 가 대체 어디있는거야?

2004년 10월에 올린 글 입니다.

참조에서 얻어온 것입니다.

가끔씩 CLASSPATH가 이리저리 어지럽게 꼬여있을 때, 도무지 내가 사용하는 클래스가 어디 붙어있는지 모를 경우가 있습니다. (사실 이것은 소스 관리가 제대로 안되고 있다는 증거라고 봅니다)
이럴경우 내가 사용하고 있는 클래스의 실제 파일이 대체 어디 있는지 알려주는 프로그램 소스가 JavaService.Net에 올라와 있어서 커맨드 라인에서 사용할 수 있도록 살짝 고쳐서 올립니다.

ClassLocation.java

import java.util.*;

public class ClassLocation {
    public static void main(String [] args) {
        if (args.length == 0) {
            usage();
            System.exit(-1);
        }

        String reqName = null;
        java.net.URL classUrl = null;

        for (int i = 0; i < args.length; i++) {
            reqName = args[i];

            reqName = reqName.replace('.', '/').trim();
            reqName = "/" + reqName + ".class";
            classUrl = ClassLocation.class.getResource(reqName);
            if (classUrl == null) {
                System.out.println(reqName + " not found");
            } else {
                System.out.println("reqName : " + classUrl.getFile());
            }
        }
    }

    public static void usage() {

        System.out.println("Usage : java ClassLocation package.classname [package.classname ....]");

    }
}

저 같은 경우 자주 겪는 현상 중의 하나가 프라퍼티 파일을 jar파일에 묶고 또 다른 클래스패스에도 두고 해서 프라퍼티 설정사항이 중복되기 때문에 제대로 적용이 안되는 현상을 겪기도 하는데요, 이럴 경우에 프라퍼티 파일이 어디 있는지도 찾아 줄 수 있도록 프로그램을 살짝 수정하여 오늘 저녁이나 내일쯤 다시 올리겠습니다.
