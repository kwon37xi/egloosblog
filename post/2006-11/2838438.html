C# 개발 환경 설정

이 글은 2005년 3월에 쓴 것임.

웬만하면 Java만 더 파고 싶은데 어쨌든 걸려오는 태클은 어쩔 수 없다.

C#을 써서 해야만 하는 작업이 있어서 .Net Framework를 설치하고 .Net Framework SDK도 설치했다.
헌데 도무지 C# 컴파일러를 찾을 수가 없었다. 당연히 SDK에 들어 있겠거니 했는데, 그게 아니라 그냥 Framework에 들어 있는 것이었다. (VS .Net 이 없으면 컴파일 조차도 못하는 줄 알고 잠깐 분노.. ^^)

저런 착각은 자바 프로그래머인 내게는 당연하다. .Net Framework의 역할을 하는 JRE에는 자바 어플리케이션을 실행하는 실행기만 있지 컴파일러가 없기 때문이다. .Net SDK 의 역할을 하는 JDK에 컴파일러가 들어있다.

현재 .Net Framework의 C#, VB .Net 등의 컴파일러가 있는 디렉토리는 "C:\WINDOWS\Microsoft.NET\Framework\v1.1.4322" 였다.

저 디렉토리와 .Net Framework SDK 의 Bin 디렉토리를 Path로 설정했다.

C# 자료를 찾으러 돌아 다니면서 참 안타까운게 있었다. 대부분 열린 읽기/쓰기 정책의 자바 개발자 그룹들과 확인히 달랐다.

대부분의 국내 C# 커뮤니티들이 상당히 폐쇄적이다. 가입을 안하면 글 읽기 조차 안되게 되어 있다.
그나마 http://www.jabook.net 이 있어서 다행이었다.

열린 마음으로 정보를 공유했으면 한다.

C# 컴파일러 csc.exe 간단 사용법

원문 : http://www.wug.or.kr/net/csc.aspx
# csc hello.cs : 컴파일
# csc /target:library /out:hello.dll hello.cs : DLL 라이브러리 작성
# csc main.cs /r:hello.dll : hello.dll를 참조하는 main.cs 컴파일
# csc /? : 도움말


VIM과 C#

* 소스코드 작성하고서 :!csc % : 컴파일
* :!%:r : 실행
